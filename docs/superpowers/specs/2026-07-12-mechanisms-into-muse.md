# 把「有趣性/张力」做进 paper-muse 的构思阶段

日期：2026-07-12　状态：**草案（待用户审定）**
关系：这是**构思幕（盲区扫描）的设计精化**，并触及卡片模型、枚举逻辑与产物契约。证据基础见[有趣性与目标实现机制](../../research/2026-07-12-interestingness-and-goal-mechanisms.md)。**与任何具体模型（Fable 等）无关**——这是项目设计层的要求，不是某个模型的提示词技巧。

## 0. 核心命题

**论文的「有趣性/张力」必须在构思那一刻就被考虑，而不是事后审稿补评。** 一张好卡不只是「相对你非显而易见的切入点」，而是「一个针对某个默认前提的反转/问题化」——它自带张力。

三条红线（不变）：不搬显化入提示词；不声称提高质量/被引；不写正稿（PRD §7）。

## 1. 张力与发现力：对齐，不是打架

先更正一处早先的误判：曾担心「有趣性 ↔ 发现力冲突」——那是针对**被引/传播力**框架（已弃）。换成**问题化/张力**后二者**对齐**：

- **发现力（核心不变式，不动）**：相对**这个研究者**（画像＋困惑）非显而易见。参照系 = 研究者。
- **张力/有趣性（本次要长进构思）**：反转/质疑**领域**视为理所当然的默认前提（Davis / Alvesson–Sandberg）。参照系 = 领域读者。

两者都是「挑战默认假设」，只是参照系不同。故张力**不是后手、不是独立一列的从属信号**，而是「好发现」的内在形状——与发现力**在同一构思阶段共同定义「值得深挖」**。

## 2. 落地：把张力做成构思的一等属性（三件事）

### (1) 卡片携带「张力」字段
每张卡除现有 `why_nonobvious`（相对研究者）外，补一个 **`tension`（相对领域默认前提）**：
- **反转了谁的哪个默认前提**——Davis 式：法学界视为理所当然的前提 P → 本卡主张修正/否定 P；或 Alvesson–Sandberg 式：常规解释所依赖的假设 A → 本卡质疑 A。
- 两个参照系都要在：`why_nonobvious`（对你新）＋ `tension`（对领域有张力）。只有前者 = 你孤陋；只有后者 = 老生常谈。**两者兼备才是金卡**。

### (2) 枚举从「空白搜寻」重定向为「问题化」
枚举任务升级：**先识别本主题下领域的默认前提，再生成反转它们的切入点**，而非罗列「其他学科/理论/方法里没人做过的角度」。
- 保留三类卡（学科视角/理论框架/研究方法），但每类都要求带 `tension`。
- 例：不问「战略性法治研究还缺什么」，而问「**常规法治观遗漏/预设了法治的哪种功能**」。

### (3) 构思阶段的质量闸（自检，非事后审稿）
在出卡前就用有趣性研究自检张力强弱：
- **Goyanes 五维**：张力至少落一类（反直觉/奠基性/新路径/示范性/实践洞见），否则疑似弱张力。
- **Corley–Gioia**：张力须**原创（改变理解）× 效用（能组织现实制度）**兼备——法学尤其要能推出具体制度。
- **Oppenheimer**：张力表述要清楚，不靠大词堆砌。

## 3. 设计要求表（构思阶段，非模型指令）

| 研究机制 | 对论文构思的要求 | 在 muse 里的体现 |
|---|---|---|
| Davis 有趣性 | 修正读者习以为常的前提 | 枚举：先列领域默认认识，再选一个**可证成**的前提反转 → 写入 `tension` |
| Alvesson–Sandberg 问题化 | 质疑既有解释的基础假设，非找空白 | 枚举重定向；卡必须指出所质疑的假设 |
| Goyanes 五维 | 反直觉/奠基/新路径/示范/实践 | 质量闸：`tension` 至少命中一维 |
| Corley–Gioia | 原创性 × 效用 | 质量闸：改变理解 **且** 能推出具体制度 |
| Oppenheimer 流畅性 | 故作复杂降分 | 质量闸：`tension`/机制表述清楚，不用大词掩盖 |
| 心理对照＋执行意图（MCII） | 目标接障碍与行动 | 产物写成「目标—障碍—if–then 验收门槛」（见 §4） |

## 4. MCII → 产物契约（执行侧，位置不变）

深挖/待办产物写成三元：**目标（理想论证）— 障碍（失败点/最强反驳）— if–then（执行动作 + 验收门槛）**，别停在「理想成稿愿景」（Kappes–Oettingen 泄劲警示）。paper-muse 已有 `steelman`/`failure-points`，此处只是显式接成执行意图。
- 「Wayfinder tickets」名称不在本仓库——若指另一工具，B 的落点归属待你确认。

## 5. 施工与顺序（重要）

- **主战场 = 构思阶段（上面 §2）**，落地要碰 `blindspot.py`（`enumerate_cards` 提示词、卡 schema、离线单测），且影响**首批 20s / 出卡率 / 生成质量**——是**核心引擎改动**，须专门切片 + TDD，**不是随手改**。

## 6. 开放问题（待审定）

1. **`tension` 必填还是软要求**？必填提高张力密度但可能压低出卡率、拖慢首批 20s；软要求则「弱张力卡」标记后仍上墙。
2. **领域默认前提从哪来**？模型自行假定，还是从研究者画像（已有立场/熟悉理论）＋困惑推导？建议后者优先（更贴参照系）。
3. **`tension` 与现有 novelty（en/zh hits）如何并置**，两个「新」不互相淹没？
4. 中文法学本地化：Goyanes 是传播学编委观点，维度语言需按法学重写。
