# 共用简报：muse 卡片墙 mock（四方向反应法）

四个 HTML mock 共用这份简报 + `cards.js` 的同一份假数据。你负责其中**一个方向**，
审美细则在你的任务提示里。这份文件是**四个方向都必须满足**的公共契约。

## 这是什么

「论文构思者」的首屏——盲区扫描**卡片墙**。帮中文法学研究者发现论文的「未知的未知」：
非显而易见的跨学科切入点、理论与方法。引擎三家模型合议出卡，每卡带新颖性判据。
本 mock 是静态视觉稿，给用户在浏览器里挑设计方向，**不接后端**（数据写死）。

最终归宿：macOS 14+ 的 **WKWebView 本地托管**（muse_server 静态托管），**无外网依赖**。

## 主题 / 画像（顶部预填）

- 主题输入框预填：`平台经济中的数据权力与法律规制`
- 研究者画像（可选，可折叠展示）：见 `cards.js` 的 `MUSE_PROFILE`
  （领域：中文法学·数据法与反垄断；立场：权利本位、法教义学训练；困惑：怕落进「数据确权」红海）

## 每个 mock 都必须包含（缺一不可）

1. **主题输入区**：一个能容一句到一段话的主题/困惑输入框 + 「扫描」触发；画像可折叠。
2. **两幕切换**：顶部 `[构思幕 | 对抗幕]`。构思幕=当前卡片墙（激活态）；**对抗幕本轮留位**
   （置灰/「即将上线」占位，别做内容）。
3. **三类卡都出现**：学科视角 / 理论框架 / 研究方法。三类要有**视觉区分**（颜色/标签/位置任选，
   但要一眼能分）。方法卡（研究方法）必须显示 `feasibility`（数据从哪来）。
4. **新颖性四档**：主流 / 边缘有人做 / 交叉空白 / 中文面未检——要能一眼读出，`交叉空白` 是金矿要最亮。
5. **三个徽标**（语义见下）：🥇 英热中冷、🔸 离群、📚 已藏未用。
   **精修成克制的印记/pill，别用生 emoji 当图标**（emoji 当区块标记是 AI 味重灾区）；
   语义不变即可，可用小色点、字符标、SVG、角标等。
6. **最强反驳**（`steelman`）：每卡必须显著呈现——这是本工具的灵魂（哪类审稿人会怎么打）。
   给它足够视觉权重，别塞进折叠里当附注。
7. **三键反馈**：每卡三个按钮 `已知` / `新但不适用` / `新且值得深挖`。
   要有**选中态**（点了变样、其余淡出/收起），`新且值得深挖` 是正向操作、视觉最重。
8. **文献锚点**（`anchors`）：`[{title,url}]`，真实检索来的，要可见（可折叠/小字，但在）。
9. **中英密度对比**：`en_hits`（英文面）vs `zh_hits`（中文学界面）要成对呈现——
   这是「英热中冷」金标的证据。`zh_hits=null` 时显示「中文面未检」并暗示解锁方式（如「需 CNKI 会话」）。
10. **拷问句**（`questions`）：1–2 句，作 grill 弹药；可放折叠区，但要在。
11. **来源模型**（`source_models`）：deepseek/gemini/openai 组合，小字标出；离群卡只有一家。
12. **流式感**：卡片入场要有轻微的错峰浮现动画（stagger fade/rise），呼应「首批卡 ≤20s 逐张上墙」。
    尊重 `prefers-reduced-motion`（关掉动画）。
13. **明暗双主题**：用 CSS 变量搭色板并提供**能用的切换按钮**（☀/☾ 或「明/暗」）。
    你的方向可以有个主基调（如仪表盘偏暗），但两套都要真的能看。

## 卡片字段速查（`cards.js` 里 7 张，别改数据）

| 字段 | 含义 | 怎么呈现 |
|---|---|---|
| `type` | 学科视角/理论框架/研究方法 | 一眼可分的三类视觉 |
| `name` | 理论/方法名 | 卡标题，排印重点 |
| `mechanism` | 一句话机制 | 副标/首段 |
| `why_nonobvious` | 为什么对**你**非显而易见 | 核心正文，点题「相对于你的未知」 |
| `steelman` | 最强反驳 | 显著区块（要素 6） |
| `feasibility` | 方法卡才有：数据从哪来 | 方法卡专属块 |
| `questions[]` | 拷问句 | 可折叠 |
| `novelty` | 新颖性四档 | 要素 4 |
| `gold` | true=🥇英热中冷 | 徽标 + 密度差高亮 |
| `outlier` | true=🔸离群（仅一家） | 徽标 |
| `own_hits` | 自有库命中；>0 且 novelty∈{交叉空白,边缘}→📚 | 徽标 + 数字 |
| `en_hits`/`zh_hits` | 英文/中文学界命中数（zh 可能 null） | 成对密度对比（要素 9） |
| `anchors[]` | 真实文献锚点 | 要素 8 |
| `source_models[]` | 来源模型 | 小字（要素 11） |

## 硬约束（四个方向共同）

- **单个自包含 HTML 文件**：所有 CSS **内联在 `<style>`**，数据内联在 `<script>`（把 `cards.js`
  的 `MUSE_CARDS/MUSE_TOPIC/MUSE_PROFILE` 抄进去），用一个小 render 函数出卡。
  **禁止任何外链**：无 CDN、无 webfont URL、无外部图片、无 fetch。整文件双击即开。
- **只用 macOS 系统字体**（WKWebView 离线，无 CDN）。按方向选，但只能从这些里取：
  - 中文衬线（宋）：`"Songti SC","STSong","Songti TC",serif`
  - 中文楷体：`"Kaiti SC","STKaiti",serif`
  - 中文黑体/苹方：`"PingFang SC","Heiti SC","Hiragino Sans GB",sans-serif`
  - 西文衬线：`"New York","Iowan Old Style","Palatino","Baskerville",Georgia,serif`
  - 西文无衬线：`-apple-system,"SF Pro Text","Helvetica Neue",sans-serif`
  - 等宽（数字/命中数用 `font-variant-numeric:tabular-nums`）：`"SF Mono","SFMono-Regular",ui-monospace,Menlo,monospace`
- **可访问性地板**：语义标签、键盘焦点可见态、`prefers-reduced-motion`、对比度够。
- **别落 AI 设计俗套**：避开 #F4F1EA 暖米+衬线+陶土、纯黑底+荧光绿/朱红点、broadsheet 发丝线密排、
  紫蓝渐变 hero、Inter/Space Grotesk 当「安全字」、emoji 当区块标记、一切居中、到处 rounded-lg、
  圆角卡+左侧色条。你的方向要**真的有观点**，把大胆花在一处、其余保持安静。
- **响应式**：宽内容（表格/长锚点）各自 `overflow-x:auto`，页面 body 不横向滚。窗口 ~900–1400px 主要目标。

## 交付

- 写到你任务提示指定的路径（每个方向一个独立文件，互不冲突）。
- 顶部留一行 `<!-- 方向名 · 一句话设计论点 -->`。
- 返回：文件路径 + 3–5 句你的设计决策（色板 hex、字体配对、布局骨架、你把「大胆」花在哪、明暗如何切）。
